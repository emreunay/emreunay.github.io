KMSENSE Motion Analyzer — User Manual

System Overview

The PCB mounts to the system under test. An MPU6050 IMU samples acceleration at 50 Hz. An STM32 MCU runs a moving-average detrend, bandpass IIR filter (0.5–6 Hz), and 256-point FFT every ~5 seconds. Results are sent over BLE via the on-board nRF52 to this dashboard.

How to Connect

  • Open this page in Chrome on Android (v85+).
  • Grant Bluetooth and Location permissions when prompted.
  • Power on the PCB and press Connect.
  • Select your device from the system Bluetooth picker.
  • Data begins arriving within one FFT window (~5 s after warm-up).

What the Metrics Mean

  • Frequency (Hz) — dominant oscillation frequency of the system (range: 1–5 Hz). Detected via FFT peak with quadratic interpolation; resolution ≈ 0.2 Hz.
  • Amplitude (mg) — peak acceleration of the oscillation (milli-g). Computed as peak-to-peak of the bandpass-filtered signal divided by 2.

Waveform Chart (live)

Shows the reconstructed signal A · sin(2π · f · t) for the last 5 FFT windows (~25 s) on a rolling basis. Wider cycles = lower frequency; taller envelope = higher amplitude.

Exporting Data

  • Export JSON — downloads a .json file with all measurements, timestamps, and summary metadata. Suitable for post-processing in Python / MATLAB.
  • Export PDF — generates a report with summary statistics, full measurement table, and stitched waveform charts (one chart per 5-window group) covering the entire session.

Packet Format

BLE notifications arrive as ASCII: freq_hz,amplitude_mg\n
Example: 2.347,83.21\n

Requirements

  • Chrome on Android (Web Bluetooth API required).
  • Page must be served over HTTPS (Netlify, GitHub Pages, etc.).
  • Bluetooth and Location permissions must be granted.
